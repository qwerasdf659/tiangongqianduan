### 问题2：today_earned / today_consumed 计算逻辑

如果 `today_summary` 字段存在但值为 0，请检查后端计算逻辑：

1. **时区问题**：计算"今日"时，是否使用了北京时间（UTC+8）？
   - 正确做法：`WHERE created_at >= '2026-02-21 00:00:00+08:00' AND created_at < '2026-02-22 00:00:00+08:00'`
   - 错误做法：`WHERE created_at >= '2026-02-21 00:00:00+00:00'`（UTC零点 ≠ 北京零点，相差8小时）
   - 如果使用 UTC 存储，查询条件应转换为：`WHERE created_at >= '2026-02-20 16:00:00+00:00'`（北京零点对应的 UTC 时间）

2. **数据源范围**：`today_earned` 是否统计了所有积分来源（不仅仅是抽奖）？
   - 积分获得来源包括：消费奖励（consumption_reward）、抽奖退还（lottery_refund）、管理员调整（admin_adjust）等
   - 积分消费来源包括：抽奖消耗（lottery_cost）、商品兑换（exchange）、市场购买（market_purchase）等
   - 如果只统计了抽奖相关的积分流水，则会遗漏其他来源

3. **SQL 查询参考**（仅作参考，请根据实际表结构调整）：

```sql
-- 今日获得（所有 direction='earn' 的交易）
SELECT COALESCE(SUM(amount), 0) AS today_earned
FROM asset_transactions
WHERE user_id = :user_id
  AND asset_code = 'POINTS'
  AND direction = 'earn'
  AND created_at >= DATE_TRUNC('day', NOW() AT TIME ZONE 'Asia/Shanghai') AT TIME ZONE 'Asia/Shanghai'
  AND created_at < DATE_TRUNC('day', NOW() AT TIME ZONE 'Asia/Shanghai') AT TIME ZONE 'Asia/Shanghai' + INTERVAL '1 day';

-- 今日消费（所有 direction='consume' 的交易）
SELECT COALESCE(SUM(amount), 0) AS today_consumed
FROM asset_transactions
WHERE user_id = :user_id
  AND asset_code = 'POINTS'
  AND direction = 'consume'
  AND created_at >= DATE_TRUNC('day', NOW() AT TIME ZONE 'Asia/Shanghai') AT TIME ZONE 'Asia/Shanghai'
  AND created_at < DATE_TRUNC('day', NOW() AT TIME ZONE 'Asia/Shanghai') AT TIME ZONE 'Asia/Shanghai' + INTERVAL '1 day';
```

### 问题3：接口路径归属是否合理

当前"今日积分趋势"挂在 `/lottery/points` 路径下（抽奖域），但它实际反映的是**所有业务类型的积分变动汇总**，与抽奖无直接关系。

**建议**（可选，由后端团队决定）：
- 方案A：保持现有路径 `/lottery/points`，确保统计范围覆盖所有 `asset_code = 'POINTS'` 的交易
- 方案B：迁移到资产域 `GET /api/v4/assets/summary`，更符合 RESTful 语义

如果采用方案B，请告知前端新的接口路径，前端会同步修改。

## 前端当前代码实现（供后端参考）

### 调用代码（pages/user/user.ts）

```typescript
async loadPointsTrend() {
  const data = await safeApiCall(() => API.getUserStatistics(), {
    context: '积分趋势',
    silent: true
  })
  if (data) {
    if (!data.today_summary) {
      log.warn('后端响应缺少 today_summary 字段，响应data结构:', Object.keys(data))
    }
    this.setData({
      todayEarned: data.today_summary?.today_earned ?? 0,
      todayConsumed: data.today_summary?.today_consumed ?? 0
    })
  } else {
    this.setData({ todayEarned: 0, todayConsumed: 0 })
  }
}
```

### API 封装（utils/api/lottery.ts）

```typescript
/** 获取当前用户综合统计 - GET /api/v4/lottery/points */
async function getUserStatistics() {
  return apiClient.request('/lottery/points', { method: 'GET', needAuth: true })
}
```
