# 小程序用户域 B2C 兑换 API — 后端需配合事项（Phase 3）

> 对齐 `docs/marketplace-stats-api-设计方案.md` §3.2、§11.3–§11.4。  
> 更新日期：2026-03-24（与小程序前端路径切换同步）

## 1. 前端已完成的契约调整（仅路径）

微信小程序所有 **用户侧 B2C 兑换** 请求，已由 **`/api/v4/backpack/exchange/*`** 统一改为 **`/api/v4/exchange/*`**。

- **实现位置**：`utils/api/backpack.ts`（`apiClient.request` 的路径参数，不含 baseUrl；完整 URL = `baseUrl` + `/api/v4` + 路径）。
- **页面与类型注释**：`packageExchange/**`、`typings/api.d.ts` 中相关注释已同步为新路径，保证「代码即文档」与数据流说明一致。
- **未改动**：背包 `/backpack`、竞价 `/backpack/bid`、C2C `/market/*`、汇率 `utils/api/market.ts` 中的 `/market/exchange-rates*`（见下文第 4 节）。

## 2. 请后端同事必须确认并反馈的信息（否则小程序会 404）

以下信息用于验证 **前后端契约一致**，请尽量以可检索形式提供（路由表、OpenAPI、或 `routes/v4` 挂载截图均可）。

| 序号 | 事项                                    | 说明                                                                                                                                                                                                                 |
| ---- | --------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| A    | **用户域是否已挂载 `/exchange` 路由树** | 与旧 `backpack/exchange` 在方法、路径片段、参数名上是否 **1:1 等价**（仅前缀变更）。                                                                                                                                 |
| B    | **端点清单**                            | 至少包含：GET `items`、`items/:exchange_item_id`；POST `/exchange`（兑换）；GET/POST `orders` 系列（列表、详情、取消、确认收货、评价、物流 track）；GET `space-stats`；GET `premium-status`；POST `unlock-premium`。 |
| C    | **认证**                                | 仍要求 `Authorization: Bearer <access_token>`；幂等类 POST 仍使用 Header `Idempotency-Key`（与现有实现一致）。                                                                                                       |
| D    | **响应格式**                            | 仍为统一 `ApiResponse`（`success` / `data` / `message` / `code` 等与现网一致）；业务字段保持 **snake_case**，前端不在此做字段映射。                                                                                  |
| E    | **样例**                                | 每个变更路径至少 **1 条脱敏 JSON 响应样例**（或指向内部 Wiki），便于前端核对字段存在性与类型。                                                                                                                       |
| F    | **发版顺序**                            | 若后端尚未上线 `/exchange`，请约定 **先后端、后小程序** 或 **双版本兼容窗口**；当前前端 **未保留** 对旧路径 `/backpack/exchange` 的兼容请求。                                                                        |

## 3. 运营端（Console）与小程序路径区分（避免误调）

- 小程序 **禁止** 调用 `/api/v4/console/*`（除非产品有单独安全设计与鉴权方案）。
- 文档中运营端物流等路径已迁至 `/api/v4/console/exchange/...`；用户端物流应对齐 **`/api/v4/exchange/orders/:order_no/track`**，与 `utils/api/backpack.ts` 中注释一致。

## 4. 仍属「产品 + 后端定稿」、前端未改动的项

| 项           | 说明                                                                                                                                              |
| ------------ | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| 用户域汇率   | 文档提议由 `/api/v4/market/exchange-rates*` 迁至 `/api/v4/assets/rates*`。**请产品定稿后**，后端书面通知前端，再改 `utils/api/market.ts` 中路径。 |
| C2C 命名空间 | `/api/v4/market/*` 是否改名为 `marketplace` 由产品评估；当前小程序仍按 **`/market/*`** 对接。                                                     |

## 5. 安全与业务边界（供产品决策）

若用户域接口返回 **运营统计、内部定价策略、未脱敏流水** 等，存在抓包泄露商业机密风险。请后端按最小权限返回；前端仅做展示与调用约定 API，**不推断后端业务规则默认值**。

---

**结论**：前端侧 Phase 3 路径切换已完成；**请后端确认用户域 `/api/v4/exchange/*` 已等价上线** 后再发布小程序体验版/正式版，避免全量 404。
