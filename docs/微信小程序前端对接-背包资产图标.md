# 微信小程序前端对接文档 — 背包/资产图标字段

> 日期：2026-05-24
> 后端修改版本：BackpackService + balance 路由图标字段补全
> 对接人：微信小程序前端开发

---

## 一、问题背景

微信小程序"我的仓库"页面中，资产文字前面没有图标、物品列表没有图片。

**根因**：后端 `BackpackService` 和 `/api/v4/assets/balances` 路由之前没有返回 `icon_url` / `image_url` 字段。

**已修复**：后端现在在以下 API 中返回图标/图片 URL。

---

## 二、涉及的 API 及新增字段

### 2.1 背包 API — `GET /api/v4/backpack`

#### assets[] 新增字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `icon_url` | `string \| null` | 资产图标完整 URL，可直接用于 `<image>` 组件的 src |

| 字段 | 类型 | 说明 |
|------|------|------|
| `image_url` | `string \| null` | 物品图片完整 URL，来源于物品模板主图（item_template.primary_media） |

## 三、前端使用方式

### 3.1 WXML 模板示例

```html
<!-- 资产列表（仓库顶部横向滚动） -->
<view class="asset-item" wx:for="{{assets}}" wx:key="asset_code">
  <image
    wx:if="{{item.icon_url}}"
    src="{{item.icon_url}}"
    mode="aspectFit"
    class="asset-icon"
  />
  <image
    wx:else
    src="/assets/images/default-asset-icon.png"
    mode="aspectFit"
    class="asset-icon"
  />
  <text class="asset-name">{{item.display_name}}</text>
  <text class="asset-amount">{{item.available_amount}}</text>
</view>

<!-- 物品列表 -->
<view class="item-card" wx:for="{{items}}" wx:key="item_id">
  <image
    wx:if="{{item.image_url}}"
    src="{{item.image_url}}"
    mode="aspectFill"
    class="item-image"
  />
  <image
    wx:else
    src="/assets/images/default-item-placeholder.png"
    mode="aspectFill"
    class="item-image"
  />
  <text class="item-name">{{item.item_name}}</text>
</view>
```

### 3.2 JS 数据获取示例

```javascript
// 获取背包数据
async loadBackpack() {
  const res = await request({
    url: '/api/v4/backpack',
    method: 'GET'
  })

  if (res.success) {
    this.setData({
      assets: res.data.assets,   // 每项包含 icon_url
      items: res.data.items      // 每项包含 image_url
    })
  }
}

// 获取余额数据（仓库顶部资产概览）
async loadBalances() {
  const res = await request({
    url: '/api/v4/assets/balances',
    method: 'GET'
  })

  if (res.success) {
    this.setData({
      balances: res.data.balances  // 每项包含 icon_url, display_name, group_code
    })
  }
}
```

## 四、注意事项

### 4.1 空值处理

- `icon_url` 和 `image_url` 可能为 `null`（运营未上传图标/图片时）
- 前端必须准备默认占位图，当值为 `null` 时显示占位图

### 4.2 域名白名单

图片 URL 域名为 `omqktqrtntnn.sealosbja.site`，需确保已在微信小程序后台 → 开发管理 → 服务器域名 → downloadFile 合法域名中配置。

### 4.3 图片代理说明

所有图片 URL 走后端代理路由 `/api/v4/images/{objectKey}`，已解决：
- Sealos 对象存储强制 `Content-Disposition: attachment` 导致小程序无法渲染的问题
- 无需在微信白名单中额外配置对象存储域名

### 4.4 当前数据状态

| 数据类型 | 图标/图片状态 |
|----------|--------------|
| 积分 (points) | ✅ 有图标 |
| 红源晶碎片 (red_core_shard) | ✅ 有图标 |
| 星石 (star_stone) | ✅ 有图标（本次修复补充） |
| 物品 (items) | ⚠️ 大部分物品模板未上传主图，`image_url` 为 null |

> 物品图片需要运营通过管理后台为物品模板上传图片后才会生效。

## 五、市场列表 API（积分商城页面）

`GET /api/v4/marketplace/listings` 已有图片字段，无需额外修改：

- 物品类挂牌：`products[].item_info.image_url`
- 资产类挂牌：`products[].asset_info.icon_url`

积分商城页面如果图标显示异常，请检查前端是否正确使用了这两个字段。

## 六、字段对照速查表

| 页面场景 | API | 图标/图片字段 |
|----------|-----|--------------|
| 仓库 - 资产概览 | `GET /api/v4/assets/balances` | `balances[].icon_url` |
| 仓库 - 资产列表 | `GET /api/v4/backpack` | `assets[].icon_url` |
| 仓库 - 物品列表 | `GET /api/v4/backpack` | `items[].image_url` |
| 积分商城 - 物品 | `GET /api/v4/marketplace/listings` | `products[].item_info.image_url` |
| 积分商城 - 资产 | `GET /api/v4/marketplace/listings` | `products[].asset_info.icon_url` |
