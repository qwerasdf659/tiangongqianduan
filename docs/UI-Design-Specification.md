## 一、产品背景与用户画像

### 1.1 产品背景

**行业痛点**：传统餐饮会员积分体系存在三个核心问题——

- 积分价值感低：用户攒了一堆积分，只能换纸巾、代金券，缺乏吸引力
- 用户粘性差：消费完就走，没有理由再打开小程序
- 缺乏社交传播：积分是孤立的，用户之间没有互动

**我们的解法**：把"积分兑换"升级为"积分游戏生态"。通过抽奖、收集、交易、DIY 创造四个核心玩法，让积分变成有趣的游戏资源，让用户有理由反复回来、主动分享。

**一句话定义**：一款围绕餐饮消费积分的游戏化微信小程序。用户到店消费获得积分，用积分抽奖获得虚拟宝石和实物奖品，宝石可在市场交易、兑换商品、或设计 DIY 饰品。

核心循环：**消费 → 积分 → 抽奖 → 收集 → 交易 / 创造**

### 1.2 目标用户画像

**主要用户：年轻女性消费者（占比约 65%）**

| 维度 | 描述 |
|------|------|
| 年龄 | 18–35 岁 |
| 性别 | 女性为主，男性为辅 |
| 职业 | 大学生、白领、年轻妈妈 |
| 消费场景 | 周末聚餐、工作日午餐、朋友约饭 |
| 手机使用习惯 | 重度微信用户，习惯小程序点餐/支付 |
| 兴趣标签 | 手工 DIY、盲盒收集、二手交易、游戏化体验 |

**用户动机分层**：

| 用户类型 | 核心动机 | 典型行为 |
|----------|----------|----------|
| 休闲型 | "吃完饭顺手抽个奖" | 低频消费，偶尔抽奖，不太交易 |
| 收集型 | "我要集齐六色源晶" | 中频消费，关注稀有度，喜欢背包整理 |
| 交易型 | "低买高卖赚星石" | 高频访问市场，关注价格走势 |
| 创造型 | "设计一条独一无二的手链" | 核心 DIY 用户，愿意花星石买材料 |

### 1.3 核心使用场景

| 场景 | 用户行为 | 设计关注点 |
|------|----------|------------|
| 到店消费后 | 出示消费码 → 获得积分 → 顺手抽奖 | 消费码要大、亮、快；抽奖入口要醒目 |
| 通勤/碎片时间 | 刷市场、看价格、挂牌交易 | 列表浏览效率、快速筛选 |
| 睡前放松 | 逛兑换商城、设计 DIY 饰品 | 沉浸感、手作温度、不催促 |
| 朋友聚餐时 | 互相展示抽到的稀有物品、分享海报 | 分享海报要好看、稀有度要有炫耀感 |
| 收到推送通知 | 物品被买走了/竞拍被超价了 | 通知文案要清晰、一步跳转到对应页面 |

### 1.5 品牌调性

- **关键词**：轻奢游戏感 · 手工匠心 · 温暖有趣 · 收集感
- **情绪**：惊喜感、收藏欲、成就感、手作温度
- **定位**：不是纯电商，不是纯游戏，而是"收集 → 交易 → 创造"的沉浸式积分生态

**设计原则**：

| 原则 | 含义 | 反面 |
|------|------|------|
| 惊喜感 | 抽奖结果要有仪式感，稀有物品要让人兴奋 | 不要平淡无感的结果展示 |
| 收集欲 | 背包、图鉴要让人想"集齐"，缺的要有引导 | 不要让用户觉得"够了" |
| 手作温度 | DIY 工坊要有手工感，不要冰冷的工业风 | 不要纯参数化的配置界面 |
| 轻量快捷 | 核心操作 3 步内完成，不要繁琐流程 | 不要多余的确认步骤 |

## 三、排版规范

### 3.1 字号（设计稿 750px 宽度）

| 用途 | 大小 |
|------|------|
| 极小（角标、标签） | 20px |
| 小（辅助文字） | 24px |
| 正文 | 28px |
| 中等（页面默认） | 32px |
| 标题 | 36px |
| 页面标题 | 40px |
| 数字高亮 | 48px |

### 3.2 字体族

系统默认字体栈，不引入自定义字体（节省包体积）：

```
font-family: -apple-system, BlinkMacSystemFont, "PingFang SC", "Helvetica Neue", "Microsoft YaHei", sans-serif;
```

### 3.3 字重与行高

| 用途 | 字重 | 行高 |
|------|------|------|
| 极小（角标、标签） | Regular 400 | 1.2 |
| 小（辅助文字） | Regular 400 | 1.4 |
| 正文 | Regular 400 | 1.5 |
| 中等（页面默认） | Medium 500 | 1.5 |
| 标题 | Semibold 600 | 1.3 |
| 页面标题 | Bold 700 | 1.3 |
| 数字高亮（价格/余额） | DIN Alternate Bold 700 | 1.2 |

> 数字场景（价格、积分、倒计时）推荐使用 DIN 或 Roboto 等等宽数字字体，保持数字对齐。

### 3.4 间距

| 级别 | 大小 |
|------|------|
| 极小 | 8px |
| 小 | 16px |
| 中 | 20px |
| 基础 | 24px |
| 大 | 32px |
| 特大 | 40px |
| 超大 | 48px |

### 3.5 圆角

| 场景 | 大小 |
|------|------|
| 按钮、输入框 | 8px |
| 卡片 | 16px |
| 弹窗 | 24px |
| 标签页 | 32px |
| 头像 | 圆形 |

### 3.6 阴影

卡片统一使用品牌色调阴影：`0 8px 20px rgba(255, 107, 53, 0.08)`，不用纯黑阴影。

### 3.7 动画

克制风格，不要花哨。

| 场景 | 时长 | 效果描述 |
|------|------|----------|
| 按钮按下/弹起 | 0.15s | 快速回弹 |
| 页面元素淡入 | 0.2s | 平滑淡入 |
| 弹窗弹出 | 0.25s | 微弹效果（略微超出再回弹） |
| 弹窗关闭 | 0.2s | 平滑收起 |
| 列表项入场 | 0.2s（逐项延迟 0.05s） | 依次滑入 |
| 抽奖跑马灯 | 见第七章 | 先快后慢，有减速感 |
| 材料飞行动画（DIY） | 0.4s | 从面板飞向画布，带弧线轨迹 |

### 3.8 主题

系统支持 6 套主题配色切换，主色会变。以 `#FF6B35` 橙色为默认主题出图即可，其他主题由开发自动适配。

> **暗黑模式**：v1.0 暂不支持暗黑模式，仅提供浅色主题。后续版本如需适配微信系统暗黑模式，需额外出一套暗色配色方案（背景、卡片、文字、分割线等）。

### 3.9 图标规范

| 属性 | 规范 |
|------|------|
| 风格 | 线性（Outline）为主，选中态可用填充（Filled） |
| 线条粗细 | 统一 2px 描边（750px 设计稿下） |
| 尺寸体系 | 小 32px / 中 40px / 大 48px / 特大 64px |
| 最小可辨识尺寸 | 不小于 32px，低于此尺寸改用文字标签 |
| 颜色 | 默认跟随文字色，选中态跟随品牌色 |
| 格式 | TabBar 和高频图标用 Iconfont 字体；资产图标（源晶/星石）用独立 SVG |
| 视觉统一 | 所有图标在同一视觉重量下保持一致，圆形图标外切圆对齐 |

## 五、页面清单与功能说明

### 5.1 首页（Tab 1）

**页面目标**：快速导航 + 运营内容展示

**内容区块（从上到下）**：

1. **顶部状态栏**
   - 左：用户头像（圆形小图）+ 昵称
   - 右：积分余额（金色数字 + 积分图标）、通知铃铛（带未读红点）

2. **轮播 Banner**（高度约屏幕 1/4）
   - 运营配置轮播图，自动轮播
   - 支持图片类型，点击可跳转小程序内页面或 WebView
   - 底部圆点指示器

3. **公告条**（Banner 下方）
   - 小喇叭图标 + 滚动文字（纯文本公告）
   - 单行横向滚动，最多 5 条

4. **快捷入口宫格**（2 行 × 4 列 = 8 个入口）
   - 抽奖、交易市场、兑换商城、DIY 工坊
   - 背包、资产转换、消费码、客服
   - 每个入口：圆形图标 + 文字标签

5. **今日推荐区**（可选）
   - 热门市场挂牌 / 新上架兑换商品 / 限时活动

**弹窗逻辑**：
- 进入首页时检查弹窗广告
- 居中展示，右上角 X 关闭
- 支持频次控制（每天一次 / 每次打开一次 / 强制展示）
- 弹窗图片可点击跳转

### 5.2 抽奖页（Tab 2 · 核心页面）

**页面目标**：游戏化抽奖体验，制造惊喜感和期待感

**内容区块**：

1. **顶部信息栏**
   - 活动名称
   - 当前积分余额（大字金色）
   - 保底进度条："距保底还有 X 次"（进度条样式，满了发光）

2. **奖品展示区**（占屏幕约 50%）
   - 默认展示模式：3×3 九宫格
   - 每个格子：奖品图片/类型图标 + 名称 + 稀有度边框颜色
   - 稀有度边框：普通灰 / 稀有绿 / 精良蓝 / 史诗紫 / 传说橙

3. **抽奖按钮区**（底部固定，4 按钮横排）
   - 「单抽」30 积分
   - 「3 连抽」90 积分
   - 「5 连抽」150 积分
   - 「10 连抽」300 积分
   - 按钮样式：金色渐变底，积分消耗小字标注
   - 积分不足时按钮置灰

4. **底部辅助入口**
   - "抽奖记录" / "规则说明" 文字链接

**12 种抽奖玩法**（每种独立游戏界面）：

| 编号 | 玩法 | 标识 |
|------|------|------|
| 1 | 转盘 | wheel |
| 2 | 九宫格 | grid |
| 3 | 刮刮卡 | scratch |
| 4 | 砸蛋 | egg |
| 5 | 扭蛋机 | gashapon |
| 6 | 盲盒 | blindbox |
| 7 | 老虎机 | slotmachine |
| 8 | 弹珠 | pinball |
| 9 | 打地鼠 | whackmole |
| 10 | 红包 | redpacket |
| 11 | 限时抢购 | flashsale |
| 12 | 卡牌收集 | cardcollect |

**抽奖动画**：
- 九宫格高亮轮转（跑马灯），最终停在中奖格
- 连抽：依次展示每个结果，或一次性翻牌展示
- 中奖结果弹窗按稀有度分级（详见第七章动效需求）

### 5.3 交易市场（Tab 3）

**页面目标**：C2C 买卖虚拟资产和物品

**内容区块**：

1. **顶部 Tab 切换**：「市场」/「我的挂牌」/「拍卖」

2. **筛选排序栏**
   - 类型筛选：全部 / 物品 / 资产
   - 排序：最新 / 价格低→高 / 价格高→低
   - 搜索框

3. **商品列表**（卡片流，每行 2 个）
   - 商品图片（资产用对应颜色宝石图标）
   - 商品名称（如"红源晶碎片 ×10"）
   - 稀有度标签（彩色小标签）
   - 价格：星石图标 + 数字
   - 卖家昵称（小字灰色）

4. **信息流广告**（穿插在列表中，带"广告"小标签）

**商品详情页**：
- 大图展示 + 商品名称 + 稀有度标签
- 价格（星石）+ 卖家信息
- 「立即购买」按钮（确认弹窗：价格 + 手续费说明）
- 价格走势图（折线图，近期成交价趋势）

**上架流程**（从背包发起）：
- 选择物品/资产 → 设定星石价格 → 确认上架
- 最多同时上架 10 件

### 5.4 背包（Tab 4）

**页面目标**：用户所有资产和物品的总览

**内容区块**：

1. **顶部资产总览卡片**（横向滚动或折叠面板）
   - 积分余额（金色大字）
   - 星石余额（带星石图标）
   - 6 色源晶碎片余额（对应颜色小图标 + 数字）
   - 6 色源晶余额
   - 每种资产显示：可用数量 / 冻结数量（冻结灰色小字）

2. **Tab 切换**：「物品」/「资产」

3. **物品列表**（卡片样式，每行 2 个）
   - 物品图片 + 名称
   - 稀有度边框颜色
   - 物品类型标签（优惠券 / 实物 / 虚拟）
   - 状态标签（可用 / 冻结中 / 已使用）
   - 来源标签（抽奖获得 / 市场购买 / DIY 铸造）

4. **资产列表**（列表样式，每行一条）

## 六、核心用户流程

### 流程 1：新用户首次使用

```
微信扫码/搜索 → 授权登录（一键获取手机号）→ 获赠 200 积分
→ 引导进入抽奖页 → 首次抽奖体验 → 获得奖品 → 引导查看背包
```

### 流程 2：到店消费获取积分

```
到店消费 → 打开「我的」→ 点击「出示消费码」→ 展示动态二维码
→ 商家扫码录入消费金额 → 系统自动计算并发放积分（推送通知）
```

### 流程 3：抽奖完整流程

```
进入抽奖页 → 查看奖品池 → 选择抽奖次数 → 确认消耗积分
→ 观看抽奖动画 → 查看结果（稀有度越高动画越华丽）
→ 奖品自动进入背包 → 可选：继续抽 / 去背包查看
```

### 流程 4：交易市场卖出

```
进入背包 → 选择物品 → 点击「上架到市场」→ 设定星石价格
→ 确认上架 → 等待买家购买 → 收到星石（扣除 5% 手续费）→ 推送通知
```

### 流程 5：交易市场买入

```
进入交易市场 → 浏览/搜索/筛选 → 点击商品查看详情
→ 点击「立即购买」→ 确认弹窗（价格 + 星石余额）
→ 支付星石 → 物品进入背包 → 推送通知
```

### 流程 6：DIY 饰品设计

```
进入 DIY 工坊 → 选择模板（手链/项链/吊坠）→ 选择尺寸
→ 进入设计画布 → 逐个位置选择材料珠子 → 实时查看费用
→ 点击预览 → 确认设计 → 支付星石 → 铸造为物品
→ 进入背包（可上架交易）
```

### 流程 7：兑换实物商品

```
进入兑换商城 → 浏览分类/搜索 → 查看商品详情 → 选择 SKU
→ 点击「立即兑换」→ 选择收货地址 → 确认扣除材料资产
→ 生成订单 → 等待发货 → 查看物流
```

### 流程 8：物品核销（到店使用）

```
进入背包 → 找到要使用的物品 → 点击「生成核销码」
→ 展示二维码 + 文字码 → 到店出示给店员
→ 店员扫码核销 → 物品状态变为「已使用」
```

## 九、特殊设计注意事项

1. **设计稿尺寸**：750px 宽度出图

2. **虚拟货币体系**：整个系统没有人民币支付，所有价格用虚拟资产标价，价格展示时使用对应资产图标

3. **虚拟资产清单**（共 16 种，需设计图标）：
   - **4 种货币**：星石、星石配额、积分、预算积分
   - **12 种材料**：6 种颜色（红/橙/黄/绿/蓝/紫）× 2 种形态（碎片/源晶）

4. **星石是核心货币**：交易市场定价、DIY 材料定价都用星石，图标需精心设计，出现频率极高

5. **源晶碎片 vs 源晶**：碎片是小块（抽奖常见奖品），源晶是完整宝石（更稀有），视觉上需明确区分

6. **Canvas 区域**：DIY 设计器的画布区域是代码动态绘制的，设计师只需设计画布外框容器和周围操作面板

7. **6 套主题**：以默认橙色主题出图即可，其他主题由开发自动适配

8. **稀有度动效**：抽奖中奖时根据物品稀有度展示不同等级特效
