## 二、纯前端已完成项（2026-06-07 落地，工具实测通过）

> 以下为纯微信小程序前端改动，不涉及 API/后端逻辑。已过 ESLint(0) + Prettier(0) + `tsc --noEmit`(0)。

### 问题三：意见反馈孤儿页 → 已补入口

- **原现象**：`packageUser/feedback/{feedback,list,detail}` 三页与 API 均已写好，但 `pages/user/user.ts` 的 `menuItems` 无"意见反馈"项，用户无任何入口可进入。
- **影响**：合规要求的"在线客服 + 意见反馈 + 我的售后"三入口缺一，`feedbacks` 表 C 端写入通道实质不可达。
- **已处置**：在 `pages/user/user.ts` 的 `menuItems` 中"我的售后"与"联系客服"之间新增"意见反馈"项（`id:'my-feedback'`，`type:'page'`，`url:'/packageUser/feedback/feedback'`，图标 `iconClass:'icon-edit'` + `tdIcon:'edit-1-filled'`，均为项目已有图标）；同步更新类注释的功能菜单列表。

### 问题五：feedback 三页 `.js` 空壳 → 已删除

- **原现象**：项目用 TS 编译插件（`useCompilerPlugins:["typescript","sass"]`）。`packageUser/feedback/` 下 `feedback/list/detail` 同时存在 `.ts` 与空壳 `.js`（`Page({ data:{} })`，无业务逻辑）；`packageUser/disputes/` 三页只有 `.ts`。
- **影响**：冗余空壳 `.js` 可能被开发者工具优先加载，造成"改了 .ts 不生效"或新旧混跑的排查困难。
- **已处置**：删除 `packageUser/feedback/{feedback,list,detail}.js` 三个空壳（经全局检索确认无 `require` 引用），统一由 `.ts` 编译，与 `disputes` 三页一致。

## 五、数据敏感（前端已遵循，列出供后端知悉，非求证）

- **`BUDGET_POINTS`（预算账户）**：前端不请求、不展示（DataSanitizer 屏蔽）。抽奖"中奖资格与消费挂钩"仅以人话规则页披露、不暴露数字。
- **资产转换方向**：前端仅提供"碎片/源晶 → 星石"单向入口，不提供反向（后端 `CONVERT_DIRECTION_FORBIDDEN` 兜底）。
- **星石计价边界**：前端按后端返回的 `cost_asset_code` 展示，不自行判断某商品能否用星石买，由后端 `AssetProductGuard` 校验（`STONE_BUY_VALUABLE_FORBIDDEN`）。
- **抽奖商业机密字段**：奖品列表/抽奖结果接口已脱敏，前端零引用 `win_probability`/`win_weight`/`stock_quantity`/`cost_points`/`max_daily_wins`。

## 六、技术框架与业务背景（对齐真实配置）

- **业务模式**：餐厅积分抽奖系统。已下线用户间(C2C)交易，改为用户↔官方(B2C)单向「道具商城」。核心域：抽奖（`packageLottery`，多种玩法）、兑换/资产转换/官方竞价（`packageExchange`，走 `exchange` + `exchange/bid` 域）、用户中心与售后（`packageUser`）。
- **技术栈**（`package.json` / `project.config.json` 实证）：微信小程序原生 + TypeScript + Sass + MobX（`mobx-miniprogram`）+ Skyline 渲染；UI 用 TDesign（`tdesign-miniprogram`）；WebSocket 用 `weapp.socket.io`；编译插件 `useCompilerPlugins:["typescript","sass"]`；版本 v5.2.0。
- **技术红线**（`.cursorrules`）：不用 mock，只对接后端真实数据；前端适配后端、字段用后端原名不做映射层；商业机密字段前端不请求不展示。
- **分包结构**（`app.json`）：`packageLottery`/`packageExchange`/`packageUser`/`packageAdmin`/`packageAd`/`packageDIY`，已无 `packageTrade`（C2C 整体下线时删除）。
