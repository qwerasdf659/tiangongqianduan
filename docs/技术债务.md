### 1.2 media_attachments 数据迁移（P0 — 影响数据正确性）

**归属**：后端数据库项目

#### ✅ 拍板结论（对齐大厂数据治理习惯）

### D2：分类标识 — 统一用 category_id ✅ 确定执行

> **一句话**：美团 / 阿里 / 腾讯 / Steam 内部全用数字 ID；没有主流平台在内部接口用字符串 `code` 做主标识。

**问题**：三个服务各自实现了 `category_code → category_id` 的翻译逻辑

#### 行业做法对比

| 公司/场景 | 分类标识方案 | 理由 |
|---|---|---|
| **美团**（商品分类） | 内部全部用数字 `category_id`，仅对外 API 暴露 `category_code` | 内部高效，外部语义化 |
| **阿里（淘宝/天猫）** | 数字 `cat_id` 贯穿前后端，`cat_name` 仅展示用 | 10 亿级商品量下，数字 ID 是唯一可行方案 |
| **腾讯游戏（道具分类）** | 数字 `type_id`，策划表用 `type_name` 做注释 | 游戏内道具用数字 ID 索引最快 |
| **Steam/网易游戏交易** | 数字 `appid` + `contextid`，不用字符串标识 | 交易系统对性能敏感 |
| **小众二手平台（闲鱼/转转）** | 数字 `category_id`，前端通过字典表翻译展示名 | 标准做法 |
| **活动策划（抽奖/积分兑换）** | 数字 ID 关联奖品池，策划后台显示名称 | 奖品类目数量少，ID 足够 |

**没有主流平台在内部接口中用 `category_code` 字符串做主标识。** 字符串标识只出现在对外开放 API（给第三方开发者用的），你的项目不需要。

#### 结论：统一用 `category_id`

你的 `categories` 表主键就是 `category_id`（数字），且只有 25 行数据。用数字 ID：
- 省一次 `WHERE code = ?` 查表
- 前后端、数据库三层一致
- 索引命中更快

**执行步骤**：
1. 在 `Category` 模型上保留 `static resolveToId(codeOrId)` 作为过渡工具方法
2. 后端三处服务统一改为只接受 `category_id`，删除 `_resolveCategoryId()` 等兼容逻辑
3. `routes/v4/console/operations/item-templates.js` L70 删除 `category_code` 兼容分支
4. admin 前端确认所有分类选择器已传 `category_id`（数字）

### D3：分页参数 — 统一用 page_size ✅ 确定执行

> **一句话**：约 90% 模块已在用 `page_size`；改 1 处（少数路由/前端）比改 N 处便宜。

**问题**：部分路由从 query 中读 `limit` 当分页参数，大部分读 `page_size`

#### 实际代码统计（实查数据）

|---|---|---|
| 后端路由文件数 | **76 个** | **3 个**（user_management、chat、market/listings） |

#### 行业做法对比

| 公司/场景 | 分页参数命名 | 备注 |
|---|---|---|
| **美团** | `page_size` + `page_no` | 标准 RESTful |
| **阿里（开放平台）** | `page_size` + `page_no` | 官方文档标准 |

| **游戏后台（腾讯/网易）** | `page_size` + `page` | 运营后台标配 |
| **虚拟物品交易平台** | `page_size` + `page` | 列表页翻页场景 |

#### 结论：在 AdminService 新增 `batchSetIndividualPrices` 方法 + 路由层修复调用

**第 1 步：在 `services/exchange/AdminService.js` 新增方法**（插入到 `batchUpdatePrice` 方法之后，约 L1676）

复用 `batchUpdatePrice` 的同套模式（SKU→渠道定价查询、审计日志、SPU 汇总刷新、缓存失效）：

```javascript
/**
 * 批量逐个设价（每个商品设不同价格）
 *
 * @param {Array<{exchange_item_id: number, cost_amount: number}>} priceItems - 商品价格数组
 * @param {Object} options - 选项
 * @param {Object} options.transaction - 事务对象（必填）
 * @param {number} [options.operator_id] - 操作人 ID
 * @returns {Promise<Object>} { affected_rows }
 */
async batchSetIndividualPrices(priceItems, options = {}) {
  const transaction = assertAndGetTransaction(options, 'AdminService.batchSetIndividualPrices')

  if (!Array.isArray(priceItems) || priceItems.length === 0) {
    throw new Error('价格数据不能为空')
  }

  const priceMap = new Map()
  for (const item of priceItems) {
    if (!item.exchange_item_id || item.cost_amount === undefined || item.cost_amount < 0) {
      continue
    }
    priceMap.set(Number(item.exchange_item_id), Number(item.cost_amount))
  }

  if (priceMap.size === 0) {
    return { affected_rows: 0 }
  }

  logger.info('[兑换市场-管理] 批量逐个设价', { count: priceMap.size })

  const ChannelPriceModel = this.models.ExchangeChannelPrice

  const skus = await this.ExchangeItemSku.findAll({
    where: { exchange_item_id: { [Op.in]: [...priceMap.keys()] } },
    include: [{
      model: ChannelPriceModel,
      as: 'channelPrices',
      where: { is_enabled: true },
      required: true
    }],
    transaction
  })

  let updatedCount = 0
  const productIdsToRefresh = new Set()

  for (const sku of skus) {
    const targetAmount = priceMap.get(sku.exchange_item_id)
    if (targetAmount === undefined) continue

    const newAmount = Math.max(1, Math.round(targetAmount))

    for (const price of sku.channelPrices) {
      const oldAmount = Number(price.cost_amount)
      if (newAmount === oldAmount) continue

      await price.update({ cost_amount: newAmount }, { transaction })
      updatedCount++
      productIdsToRefresh.add(sku.exchange_item_id)

      try {
        await this.models.AdminOperationLog.create(
          {
            operator_id: options.operator_id || null,
            operation_type: 'price_change',
            target_type: 'product',
            target_id: sku.exchange_item_id,
            action: 'batch_set_individual',
            before_data: { cost_amount: oldAmount, sku_id: sku.sku_id },
            after_data: { cost_amount: newAmount, sku_id: sku.sku_id },
            changed_fields: { cost_amount: { from: oldAmount, to: newAmount } },
            reason: '批量逐个设价',
            risk_level: 'medium',
            requires_approval: false,
            approval_status: 'not_required'
          },
          { transaction }
        )
      } catch (logErr) {
        logger.warn('[兑换市场] 价格变动日志记录失败', { error: logErr.message })
      }
    }
  }

  for (const pid of productIdsToRefresh) {
    await this._updateSpuSummary(pid, transaction)
  }

  await BusinessCacheHelper.invalidateExchangeItems('batch_individual_price_set').catch(() => {})

  logger.info('[兑换市场-管理] 批量逐个设价完成', {
    requested: priceMap.size,
    affected: updatedCount
  })
  return { affected_rows: updatedCount }
}
```
