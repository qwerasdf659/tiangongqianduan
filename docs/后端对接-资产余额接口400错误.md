## 四、需要后端确认/提供的信息

### 4.1 核心问题（必须回复）

1. **`asset_code` 的合法值列表是什么？**
   - 前端当前使用 `POINTS`（大写）
   - 后端期望的值是什么？（如 `points` 小写？`POINT` 单数？其他编码？）
   - 请提供数据库 `assets` 或 `asset_types` 表中所有已配置的 `asset_code` 值

2. **`asset_code` 校验规则是大小写敏感还是不敏感？**
   - 即 `POINTS` 和 `points` 是否等价？

3. **数据库是否已初始化（seed）资产类型数据？**
   - `asset_types` 表或等效配置中是否存在 `POINTS` 记录？

### 4.2 接口契约确认（重要）

请确认以下接口契约，前端将严格按照后端返回格式编写代码：

**GET /api/v4/assets/balance**

| 参数 | 位置 | 类型 | 必填 | 说明 |
|------|------|------|------|------|
| asset_code | query | string | 是 | 资产类型编码，前端当前传 `POINTS` |

**期望响应 (HTTP 200):**
```json
{
  "success": true,
  "code": "xxx",
  "message": "xxx",
  "data": {
    "asset_code": "???",
    "available_amount": 0,
    "frozen_amount": 0,
    "total_amount": 0
  }
}
```

请确认:
- `data` 中的字段名是否仍为 `available_amount` / `frozen_amount` / `total_amount`？
- 金额字段类型是 `number`（整数）还是 `string`（字符串数字）？

### 4.3 相关接口确认

以下接口同样使用 `asset_code` 参数，请一并确认合法值：

| 接口路径 | 方法 | 前端传值 |
|----------|------|----------|
| `/api/v4/assets/balance` | GET | `asset_code=POINTS` |
| `/api/v4/assets/transactions` | GET | `asset_code=POINTS`（可选） |
| `/api/v4/assets/today-summary` | GET | `asset_code=POINTS` |
