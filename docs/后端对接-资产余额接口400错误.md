> 提交日期: 2026-03-17
> 修复日期: 2026-03-17
> 状态: **已修复** ✅
> 修复方式: 数据库迁移 `20260317010022-reenable-points-asset-type.js`

## 一、问题原因与修复说明

### 根因

数据库迁移 `20260314204916-disable-points-asset-type.js` 错误地将 POINTS 资产类型的 `is_enabled` 设为 `0`（禁用），路由校验逻辑检测到 `is_enabled === false` 后返回了 400。

```
前端请求: GET /api/v4/assets/balance?asset_code=POINTS
    ↓
路由校验: MaterialAssetType.findOne({ asset_code: 'POINTS' })
    ↓
查询结果: { is_enabled: 0 }  ← 被错误禁用
    ↓
返回 400: "无效的资产类型"
```

### 修复内容

通过新迁移将 POINTS 恢复为启用状态：

| 字段 | 修复前 | 修复后 |
|------|--------|--------|
| is_enabled | 0（禁用） | 1（启用） |
| form | currency | currency（不变） |
| display_name | 普通积分 | 积分 |

## 二、接口契约确认（前端可直接使用）

### 2.1 GET /api/v4/assets/balance - 查询单个资产余额

**请求：**

| 参数 | 位置 | 类型 | 必填 | 说明 |
|------|------|------|------|------|
| asset_code | query | string | 是 | 资产类型编码 |
| Authorization | header | string | 是 | `Bearer <JWT Token>` |

**成功响应 (HTTP 200)：**

```json
{
  "success": true,
  "code": "SUCCESS",
  "message": "操作成功",
  "data": {
    "asset_code": "POINTS",
    "available_amount": 1499940,
    "frozen_amount": 8120,
    "total_amount": 1508060
  },
  "timestamp": "2026-03-17T01:01:12.354+08:00",
  "version": "v4.0",
  "request_id": "req_xxx"
}
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| asset_code | string | 资产代码 |
| available_amount | number（整数） | 可用余额 |
| frozen_amount | number（整数） | 冻结金额 |
| total_amount | number（整数） | 总额（= available + frozen） |

**错误响应：**

| HTTP状态码 | code | 场景 |
|-----------|------|------|
| 400 | BAD_REQUEST | 缺少 asset_code 参数 |
| 400 | BAD_REQUEST | 无效的 asset_code（不存在/已禁用/内部配额类型） |
| 401 | UNAUTHORIZED | 未携带或无效的 JWT Token |

## 三、asset_code 合法值列表

### 前端可查询的资产类型（is_enabled=1 且 form≠quota）

| asset_code | display_name | form | is_tradable | 说明 |
|-----------|-------------|------|------------|------|
| `POINTS` | 积分 | currency | 否 | 用户积分，抽奖消耗/获得 |
| `DIAMOND` | 钻石 | currency | 是 | 高价值货币，可交易 |
| `red_shard` | 红水晶碎片 | shard | 是 | 碎片类材料 |
| `red_crystal` | 红水晶 | crystal | 是 | 水晶类材料 |
| `orange_shard` | 橙水晶碎片 | shard | 是 | 碎片类材料 |
| `orange_crystal` | 橙水晶 | crystal | 是 | 水晶类材料 |
| `yellow_shard` | 黄水晶碎片 | shard | 是 | 碎片类材料 |
| `yellow_crystal` | 黄水晶 | crystal | 是 | 水晶类材料 |
| `green_shard` | 绿水晶碎片 | shard | 是 | 碎片类材料 |
| `green_crystal` | 绿水晶 | crystal | 是 | 水晶类材料 |
| `blue_shard` | 蓝水晶碎片 | shard | 是 | 碎片类材料 |
| `blue_crystal` | 蓝水晶 | crystal | 是 | 水晶类材料 |
| `purple_shard` | 紫水晶碎片 | shard | 是 | 碎片类材料 |
| `purple_crystal` | 紫水晶 | crystal | 是 | 水晶类材料 |

### 前端禁止查询的资产类型

| asset_code | 原因 |
|-----------|------|
| `BUDGET_POINTS` | 系统内部预算积分，被路由层硬编码拦截 |
| 任何 form='quota' 的资产 | 内部配额类资产，路由层动态拦截 |

### asset_code 校验规则

- **大小写敏感**：`POINTS` 和 `points` 不等价，必须精确匹配
- **校验顺序**：① 非空检查 → ② 黑名单检查 → ③ 数据库存在性检查 → ④ form≠quota → ⑤ is_enabled=true

## 四、前端对接要点

1. **积分余额**：`asset_code=POINTS`（大写）
2. **金额类型**：所有金额字段均为 `number`（整数），非字符串
3. **余额为零**：用户从未持有某资产时，返回 `{ available_amount: 0, frozen_amount: 0, total_amount: 0 }`（HTTP 200，不是 404）
4. **时间格式**：`timestamp` 字段为北京时间 ISO 8601 格式（`+08:00`）
5. **交易流水时间**：`created_at` 为北京时间格式 `YYYY-MM-DD HH:mm:ss`
