# 后端需求文档：未登录浏览 + 登录弹窗优化

**文档日期**: 2026-05-14  
**需求来源**: 前端小程序 UI/UX 优化  
**优先级**: 高  
**影响范围**: 认证模块、活动接口、商城配置接口

---

## 一、需求背景

当前小程序各页面在用户未登录时，大部分功能不可用或显示空白状态。为提升用户体验和转化率，需要实现"先浏览后登录"模式：

- 用户未登录时可以浏览内容（活动列表、商品列表、DIY 设计等）
- 用户点击交互按钮（参与活动、兑换商品、保存设计等）时弹出登录弹窗
- 登录方式支持：微信一键登录 + 手机验证码登录

---

## 二、接口变更需求

### 2.1 活动列表接口 — 取消认证要求

**接口**: `GET /api/v4/lottery/campaigns/active`

**当前状态**: 需要 Bearer Token 认证（needAuth: true）

**需求变更**: 改为**无需认证**即可访问（公开接口）

**原因**: 发现页面（Tab 2）需要在未登录状态下展示进行中的活动列表，吸引用户登录参与。

**注意事项**:
- 返回数据不变：campaign_code, campaign_name, campaign_type, status, start_time, end_time, is_featured, display_tags 等
- 不涉及用户个人数据，公开展示无安全风险
- 如果后端有中间件统一拦截未认证请求，需要将此路由加入白名单

---

### 2.2 商城配置接口 — 确认无需认证（已满足）

以下接口当前已经是 `needAuth: false`，无需修改，仅确认：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v4/system/config/exchange-page` | GET | 兑换页面配置 |
| `/api/v4/system/config/product-filter` | GET | 商品筛选配置 |

---

### 2.3 验证码登录接口 — 参数格式确认

**接口**: `POST /api/v4/auth/login`

**请求体格式**:
```json
{
  "mobile": "13612227930",
  "verification_code": "123456",
  "platform": "wechat_mp"
}
```

**前端已修复的 Bug**:
- 之前前端调用时参数传递格式错误（传了对象而非独立参数），导致后端收到 `verification_code: undefined`
- 已修复为正确的参数传递方式

**后端确认事项**:
- 验证码登录时，`verification_code` 字段为必填，后端返回 `VERIFICATION_CODE_REQUIRED` 错误码是正确行为
- 开发阶段万能验证码 `123456` 是否仍然有效？

## 三、安全考量

### 3.1 公开接口限流

将 `/api/v4/lottery/campaigns/active` 改为公开接口后，建议：

- 添加 IP 级别限流（如 60次/分钟）
- 返回数据不包含任何用户敏感信息
- 仅返回 status=active 的活动，不暴露已结束/未开始的活动详情

### 3.2 登录态校验

以下操作仍然**必须**验证 Bearer Token：

| 操作 | 接口 | 说明 |
|------|------|------|
| 参与抽奖 | POST /api/v4/lottery/draw | 消耗积分 |
| 兑换商品 | POST /api/v4/exchange/redeem | 消耗积分 |
| 保存 DIY 设计 | POST /api/v4/diy/works | 写入用户数据 |
| 查看积分明细 | GET /api/v4/points/history | 用户隐私数据 |
| 查看订单 | GET /api/v4/orders | 用户隐私数据 |

## 六、测试验证要点

1. **未登录访问活动列表**: 不带 Token 请求 `GET /api/v4/lottery/campaigns/active`，应返回 200 + 活动数据
2. **验证码登录**: 使用万能验证码 `123456` + 有效手机号，应返回 access_token
3. **已登录跨页面同步**: 在任意页面登录成功后，切换到其他 Tab 页面，登录弹窗应自动关闭
4. **未登录操作拦截**: 未带 Token 请求 `POST /api/v4/lottery/draw` 等写操作接口，应返回 401
