# 奖品预览区闪烁问题 — 前端修复方案

> **发现日期**: 2026-03-10  
> **严重等级**: 中（体验缺陷）  
> **影响范围**: 所有抽奖玩法的奖品池预览区（金蛋/转盘/九宫格等底部奖品横排列表）  
> **修复方**: 前端  
> **问题分类**: 微信小程序前端渲染性能问题  

---

## 一、问题描述

用户在抽奖页面操作过程中，奖品预览区（底部横向滚动的奖品列表）出现 **短暂空白后又重新显示** 的闪烁现象。

### 触发场景

| 场景 | 触发频率 | 说明 |
|---|---|---|
| 切换到其他Tab后切回"抽奖"Tab | 每次必现 | `onShow()` 触发级联重渲染 |
| 下拉刷新 | 偶发 | `onPullDownRefresh()` 触发 `_loadCampaigns()` |
| 抽奖完成后关闭结果弹窗 | 偶发 | `_syncPointsBalance()` 触发组件更新 |

### 用户感知

奖品预览区（含跑马灯动画）在不到 1 秒内出现"消失 → 重新出现"的闪烁，尤其在低端安卓设备上更为明显。

---

## 二、这不是后端问题

| 排查维度 | 结果 |
|---|---|
| 后端是否发送了空数据再发真数据？ | **否** — API 只响应一次，数据完整 |
| 后端是否存在数据延迟或二次推送？ | **否** — 奖品列表接口无 WebSocket 推送，纯 HTTP GET |
| 是否 mock 数据引起？ | **否** — 全项目扫描无任何业务 mock 数据 |
| Token 认证是否异常导致重新请求？ | **否** — Tab 切换时 Token 有效，不触发刷新 |

### 核心结论

> 后端数据正常且只发送一次。问题出在前端：**不必要的 `setData` 调用触发了组件级联重渲染**，导致 CSS 动画中断和 `<image>` 组件白闪。

---

## 三、根本原因（前端）

### 问题数据流

```
用户切Tab回到抽奖页面
         │
         ▼
lottery.ts  onShow()
         │
         ├─► setData({ loading: false })                    ← 第1次页面重渲染
         ├─► setData({ isLoggedIn, pointsBalance, userInfo })  ← 第2次页面重渲染
         ├─► _loadCampaigns()
         │      └─► API 返回后 setData({ extraCampaigns })  ← 第3次页面重渲染
         │
         ▼
lottery-activity 组件  pageLifetimes.show()
         │
         ▼
_syncPointsBalance()
         │
         ▼
this.setData({ pointsBalance })  ← 🔴 问题根源：每次都调用，即使值未变化！
         │
         ▼
egg 子组件接收 pointsBalance 属性更新
         │
         ▼
WeChat 框架对 egg 组件整体执行 virtual-DOM diff
         │
         ├─► 奖品预览区 CSS marquee 动画帧被中断       → 用户看到短暂空白
         ├─► <image> 组件在 diff 过程中短暂失去渲染     → 图片白闪
         └─► <rarity-effects> 深层嵌套放大了闪烁效果    → 每个奖品项都闪一次
         │
         ▼
    ┌──────────────────────────────────────┐
    │  用户看到：奖品短暂空白 → 又出现     │
    └──────────────────────────────────────┘
```

### 本质原因

微信小程序的 `setData` 是昂贵操作 —— **即使设置相同的值，框架依然会触发组件树 diff 和重渲染**。值比较是开发者的责任，框架不会自动跳过。

三个问题点叠加造成了可见闪烁：

| 问题点 | 位置 | 影响 |
|---|---|---|
| `_syncPointsBalance()` 无条件调 `setData` | `lottery-activity.ts:503-510` | 每次 Tab 切换都触发 egg 组件重渲染 |
| `onShow()` 多次独立 `setData` | `lottery.ts:183, 193` | 同一生命周期内 3+ 次页面级渲染，级联传导到子组件 |
| `_loadCampaigns()` 总是更新 `extraCampaigns` | `lottery.ts:504` | 即使内容不变也替换数组引用，触发页面 WXML 重新求值 |

## 四、修复方案（已实施）

### 修复 1：`_syncPointsBalance()` 增加值比较守卫

**文件**: `packageLottery/lottery-activity/lottery-activity.ts`

```
修改前（每次都触发 setData）:
_syncPointsBalance() {
  const pointsBalance = pointsStore.availableAmount || 0
  this.setData({ pointsBalance })        ← 即使值相同也调用
}

修改后（仅值变化时触发）:
_syncPointsBalance() {
  const latestBalance = pointsStore.availableAmount || 0
  if (latestBalance !== this.data.pointsBalance) {
    this.setData({ pointsBalance: latestBalance })
  }
}
```

### 修复 2：`onShow()` 合并为单次 setData

**文件**: `pages/lottery/lottery.ts`

```
修改前（3次独立 setData）:
onShow() {
  this.setData({ loading: false })                                // 第1次
  this.setData({ isLoggedIn: true, pointsBalance, userInfo })     // 第2次
  // ... _loadCampaigns() 内部还有第3次
}

修改后（合并为1次，仅包含变化字段）:
onShow() {
  this.setData({ loading: false })
  const showBatchUpdate = {}
  if (!this.data.isLoggedIn) showBatchUpdate.isLoggedIn = true
  if (currentBalance !== this.data.pointsBalance) showBatchUpdate.pointsBalance = currentBalance
  if (userInfo !== this.data.userInfo) showBatchUpdate.userInfo = userInfo
  if (Object.keys(showBatchUpdate).length > 0) this.setData(showBatchUpdate)
}
```

### 修复 3：`_loadCampaigns()` 跳过内容相同的 extraCampaigns 更新

**文件**: `pages/lottery/lottery.ts`

```
修改前（总是替换数组）:
this.setData({ extraCampaigns: processedResult.extraCampaigns })

修改后（内容相同时跳过）:
const existingExtraCodes = this.data.extraCampaigns.map(c => c.campaign_code).join(',')
const newExtraCodes = processedResult.extraCampaigns.map(c => c.campaign_code).join(',')
if (existingExtraCodes !== newExtraCodes) {
  this.setData({ extraCampaigns: processedResult.extraCampaigns })
}
```

### 修复 4：CSS 合成层隔离

**文件**: `packageLottery/lottery-activity/sub/egg/egg.scss`

```scss
/* 跑马灯轨道 — 提升为独立合成层，隔离父组件重渲染的波及 */
.egg-prizes__track--animate {
  animation: eggPrizeMarquee linear infinite;
  will-change: transform;
  transform: translateZ(0);           /* 新增：独立合成层 */
  backface-visibility: hidden;        /* 新增：消除合成层闪烁 */
}

/* 奖品图片 — 固定尺寸+合成层，防止 layout shift 白闪 */
.egg-prizes__image {
  width: 64rpx;
  height: 64rpx;
  min-width: 64rpx;                   /* 新增：防止尺寸塌陷 */
  min-height: 64rpx;                  /* 新增：防止尺寸塌陷 */
  transform: translateZ(0);           /* 新增：独立合成层 */
}
```
