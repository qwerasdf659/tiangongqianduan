# 奖品预览区闪烁问题 — 前端修复方案

> **发现日期**: 2026-03-10  
> **严重等级**: 中（体验缺陷）  
> **影响范围**: 所有抽奖玩法的奖品池预览区（金蛋/转盘/九宫格等底部奖品横排列表）  
> **修复方**: 前端  
> **问题分类**: 微信小程序前端渲染性能问题  
> **最新修复日期**: 2026-03-14  

---

## 一、问题描述

用户在抽奖页面操作过程中，奖品预览区（底部横向滚动的奖品列表）出现 **短暂空白后又重新显示** 的闪烁现象。

### 触发场景

| 场景 | 触发频率 | 说明 |
|---|---|---|
| 切换到其他Tab后切回"抽奖"Tab | 每次必现 | `onShow()` 触发级联重渲染 |
| 下拉刷新 | 偶发 | `onPullDownRefresh()` 触发 `_loadCampaigns()` |
| 抽奖完成后关闭结果弹窗 | 偶发 | `_syncPointsBalance()` 触发组件更新 |

### 用户感知

奖品预览区（含跑马灯动画）在不到 1 秒内出现"消失 → 重新出现"的闪烁，尤其在低端安卓设备上更为明显。

---

## 二、这不是后端问题

| 排查维度 | 结果 |
|---|---|
| 后端是否发送了空数据再发真数据？ | **否** — API 只响应一次，数据完整 |
| 后端是否存在数据延迟或二次推送？ | **否** — 奖品列表接口无 WebSocket 推送，纯 HTTP GET |
| 是否 mock 数据引起？ | **否** — 全项目扫描无任何业务 mock 数据 |
| Token 认证是否异常导致重新请求？ | **否** — Tab 切换时 Token 有效，不触发刷新 |

### 核心结论

> 后端数据正常且只发送一次。问题出在前端：**过量的 `setData` 调用触发了页面级 WXML 重复求值和组件级联重渲染**，导致 CSS 动画帧被中断和 `<image>` 组件白闪。

---

## 三、根本原因（前端）

### 问题数据流（深层分析 — 2026-03-14）

```
用户切Tab回到抽奖页 onShow()
         │
         ├──► setData({ loading: false })                    ← 即使已经是 false 也调用 #1
         ├──► setData(showBatchUpdate)                       ← 独立 setData #2
         ├──► setData({ globalThemeStyle })                  ← 独立 setData #3
         ├──► loadConsumptionRecordsCount() → setData        ← 异步独立 setData #4
         ├──► loadInventoryItemCount() → setData             ← 异步独立 setData #5
         ├──► loadNotificationUnreadCount() → setData        ← 异步独立 setData #6
         ├──► setData({ qrCountdown, qrExpired })            ← 独立 setData #7
         ├──► _refreshPoints() → MobX → setData             ← 异步 setData #8
         └──► _loadCampaigns() → setData                     ← 异步 setData #9
         │
         ▼
    ┌──────────────────────────────────────────────────┐
    │  9 次 setData = 9 次页面级 WXML 完整重新求值     │
    │  每次都触发 lottery-activity → egg 组件级联 diff  │
    │  在 ~600ms 窗口内，CSS marquee 动画帧被打断      │
    │  → 用户看到奖品预览区闪烁                        │
    └──────────────────────────────────────────────────┘
```

### 本质原因

微信小程序的 `setData` 是昂贵操作 — 每次调用都会触发页面级 WXML 完整重新求值和组件树 virtual-DOM diff。即使组件属性未变化，框架仍需遍历比较。在 glass-easel 框架下，单次 diff 耗时约 5-15ms，9 次累计 50-130ms 主线程阻塞，导致 60fps CSS 动画（16.7ms/帧）出现可见掉帧。

**两层问题叠加：**

| 层级 | 问题点 | 影响 |
|---|---|---|
| 页面层 `lottery.ts` | `onShow()` 中 4 处同步 setData + 3 个异步回调各自独立 setData | 同一生命周期内 9 次页面级 WXML 求值 |
| 组件层 `lottery-activity.ts` | `_syncPointsBalance()` 无条件调 setData（已在 v1 修复中加守卫） | 每次 Tab 切换触发 egg 组件重渲染 |
| CSS 层 `egg.scss` | 缺少 `isolation` 和 `contain` 渲染隔离 | 父级重绘波及跑马灯合成层 |

### 第二轮修复（2026-03-14 — 页面层 setData 总量收敛）

> **v1 修复后问题仍存在的原因**：v1 仅解决了组件层的无条件 setData，但页面层 `onShow()` 仍有 9 次独立 setData 调用，每次都触发页面级 WXML 完整重新求值，累计主线程阻塞仍然足以打断 CSS 动画帧。

#### 修复 5：`onShow()` 所有同步 setData 合并为单次调用

**文件**: `pages/lottery/lottery.ts`

```
修改前（4处独立同步 setData）:
onShow() {
  this.setData({ loading: false })                                  // #1
  this.setData(showBatchUpdate)                                     // #2
  this.setData({ globalThemeStyle })                                // #3
  this.setData({ qrCountdown: 0, qrExpired: true, ... })           // #4
}

修改后（合并为单次 onShowPatch）:
onShow() {
  const onShowPatch = {}
  if (this.data.loading) onShowPatch.loading = false                // 守卫：已false不设
  if (!this.data.isLoggedIn) onShowPatch.isLoggedIn = true
  if (balance changed) onShowPatch.pointsBalance = ...
  if (theme changed) onShowPatch.globalThemeStyle = ...
  if (qr expired) onShowPatch.qrCountdown = 0, ...
  if (Object.keys(onShowPatch).length > 0) this.setData(onShowPatch)  // 单次调用
}
```

#### 修复 6：角标计数并行加载 + 合并为单次 setData

**文件**: `pages/lottery/lottery.ts`

```
修改前（3个独立异步 setData）:
await this.loadConsumptionRecordsCount()   // → setData({ auditRecordsCount })
this.loadInventoryItemCount()              // → setData({ inventoryItemCount })
this.loadNotificationUnreadCount()         // → setData({ notificationUnreadCount })

修改后（Promise.all 并行 + 单次 setData）:
async _refreshBadgeCounts() {
  const [records, stats, notify] = await Promise.all([...])
  const badgePatch = {}
  if (count changed) badgePatch.auditRecordsCount = ...
  if (count changed) badgePatch.inventoryItemCount = ...
  if (count changed) badgePatch.notificationUnreadCount = ...
  if (Object.keys(badgePatch).length > 0) this.setData(badgePatch)  // 单次调用
}
```

#### 修复 7：CSS 渲染隔离增强

**文件**: `packageLottery/lottery-activity/sub/egg/egg.scss`

```scss
.egg-prizes {
```
