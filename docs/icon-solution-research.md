## 各类公司怎么做的

### 大厂（美团/腾讯/阿里）

| 方案 | 做法 | 特点 |
|------|------|------|
| 通用图标 | 自研设计系统图标库（TDesign/Ant Design/Semi Design） | 专业设计师团队维护，统一规范 |
| 业务图标 | 各业务线设计师按规范画，上传到内部 iconfont 平台 | 有严格的设计规范文档 |
| 游戏/活动图标 | 运营设计师单独画，用 PNG/SVG 图片引入 | 不走通用图标库，每次活动单独出 |
| 管理方式 | 内部 iconfont 平台（类似阿里 iconfont.cn）统一管理 | 版本控制、权限管理、自动生成代码 |

### 游戏公司（米哈游/网易游戏/莉莉丝）

| 方案 | 做法 |
|------|------|
| 游戏内图标 | 全部用 PNG/WebP 切图，设计师逐个精绘 |
| 尺寸 | 通常 2x/3x 多倍图，或 SVG 矢量 |
| 风格 | 高度定制化，与游戏世界观一致 |
| 管理 | 资源管理系统，按模块/版本分类 |

### 活动策划公司（有赞/微盟/凡科）

| 方案 | 做法 |
|------|------|
| 通用 UI | 用组件库自带图标（Vant/TDesign） |
| 活动图标 | 设计师画好后切 PNG，或用 Lottie 动画 |
| 游戏玩法 | 每个玩法模板配套一组图标素材包 |

### 虚拟物品交易/二手平台（得物/转转/闲鱼）

| 方案 | 做法 |
|------|------|
| 通用图标 | 自研图标库或 Ant Design 图标 |
| 商品分类图标 | 设计师画的彩色 SVG/PNG |
| 2025趋势 | 闲鱼已转向 3D 图标 + 微动画（Lottie） |
| 品牌感 | 图标有独特的品牌风格，不用通用库 |

### 小公司/独立开发者

| 方案 | 做法 |
|------|------|
| 预算有限 | 直接用 TDesign/Vant 自带图标 |
| 稍有预算 | 阿里 iconfont.cn 找免费图标打包 |
| 有设计师 | 设计师画一套，生成 iconfont |
| 最省事 | emoji 直接用（你之前的方案） |

## 五种可选方案对比

### 方案 A：TDesign 图标 + 自建 iconfont 补充（当前方案）

```
通用图标 → TDesign t-icon（精致）
游戏图标 → 自建 tiangong-icons iconfont（粗糙）
Tab 栏 → 独立 SVG 文件
```

- 优点：改动最小，通用图标已经精致
- 缺点：三套方案混用，游戏图标质量差，维护混乱
- 长期维护成本：高（三套体系要同时维护）
- 技术债务：中等

### 方案 B：全站统一用 TDesign + game-icons.net 补充

```
通用图标 → TDesign t-icon
游戏图标 → game-icons.net 开源 SVG（CC 3.0 协议，需署名）
统一通过 <t-icon> 或 <image> 引入
```

- 优点：图标质量统一较高，开源免费
- 缺点：game-icons 风格偏欧美桌游，与中国风品牌不搭
- 长期维护成本：中
- 技术债务：低

### 方案 C：全站统一用 IconPark（字节跳动）

```
所有图标 → IconPark SVG（2000+ 图标，含游戏类）
导出为 SVG 文件，通过 <image> 组件引入
支持线性/填充/双色/四色 四种风格切换
```

- 优点：图标数量多、风格统一、支持多色、开源免费（Apache 2.0）
- 缺点：没有微信小程序专用组件，需要自己封装；游戏专属图标仍然有限
- 长期维护成本：低
- 技术债务：低

### 方案 D：阿里 iconfont.cn 自选图标打包（推荐）

```
所有图标 → 在 iconfont.cn 上搜索挑选精致图标
打包为一个 woff2 字体文件
通过 .iconfont 类名引入（和现在一样的方式）
```

- 优点：
  - 图标数量极多（百万级），游戏/抽奖/交易类全覆盖
  - 可以逐个挑选最精致的图标，不受限于单一设计师风格
  - 打包为单一字体文件，性能最优
  - 与现有 iconfont 方案完全兼容，零迁移成本
  - 支持在线预览、调整大小
- 缺点：
  - 只支持单色
  - 不同作者的图标风格可能不统一（需要人工筛选同风格的）
  - 需要注册账号、手动管理项目
- 长期维护成本：低
- 技术债务：最低（与现有架构完全一致）

### 方案 E：找设计师定制一套（最高品质）

```
所有图标 → 设计师按品牌规范统一绘制
输出 SVG，生成 iconfont 或直接用 SVG
```

- 优点：品质最高、风格完全统一、品牌感最强
- 缺点：成本高（50-100 个图标约 2000-5000 元）、周期长（1-2 周）
- 长期维护成本：低（一次性投入）
- 技术债务：最低

## 最适合你项目的方案：方案 D（阿里 iconfont.cn）

### 为什么选 D

1. **与现有技术栈完全兼容**
   - 你项目已经有 `tiangong-icons` iconfont 体系
   - SCSS 中的 `.iconfont` + `.icon-xxx::before` 写法不用改
   - 只需要替换 woff2 字体文件和 codepoint 映射

2. **零迁移成本**
   - 不需要改 WXML 结构（不用把 `<text class="iconfont">` 改成 `<t-icon>`）
   - 不需要给每个组件注册新组件
   - 不需要处理 `styleIsolation` 问题

3. **游戏图标全覆盖**
   - iconfont.cn 上有大量游戏/抽奖/交易类图标
   - 搜索"骰子""老虎机""扑克""奖杯""红包""金币"都能找到精致的
   - 可以挑选同一个设计师/同一套风格的图标，保证统一

4. **Skyline 兼容性最好**
   - iconfont 是纯字体渲染，Skyline 完美支持
   - 不存在 SVG 在 Skyline 下模糊/报错的问题
   - 不存在 `<image>` 组件加载 SVG 的兼容性坑

5. **性能最优**
   - 单一 woff2 文件，一次加载全部图标
   - 比 `<t-icon>` 组件少一层 WXML 节点
   - 比 `<image src="xxx.svg">` 少 HTTP 请求

6. **项目还没上线，可以一次性替换**
   - 不用兼容旧接口
   - 直接在 iconfont.cn 上建项目，挑选 51 个精致图标
   - 下载字体文件替换 `src-icons/output/tiangong-icons.woff2`
   - 更新 `_iconfont.scss` 中的 codepoint

### 具体执行步骤

1. 注册/登录 iconfont.cn
2. 创建项目"天工前端"
3. 搜索并添加 51 个图标（按现有图标名逐个找对应的精致版本）
4. 统一设置：Font Family = tiangong-icons，前缀 = icon-
5. 下载字体包（选 woff2 格式）
6. 替换 `styles/_iconfont.scss` 中的 base64 和 codepoint
7. 删除 `src-icons/` 目录（不再需要自己画 SVG）
8. 完成

### 关于 Tab 栏

Tab 栏的 5 个 SVG 图标（wheel/store/compass/tag/user）质量已经不错了，可以保留。或者也从 iconfont.cn 下载对应的 SVG 替换到 `custom-tab-bar/icons/` 目录。

## 结论

**方案 D（阿里 iconfont.cn）是你项目的最优解。**

理由：与现有技术架构完全一致、零迁移成本、游戏图标全覆盖、Skyline 兼容性最好、长期维护成本最低。唯一需要花的时间是在 iconfont.cn 上挑选 51 个风格统一的精致图标（约 1-2 小时）。

如果预算允许且追求极致品牌感，可以在方案 D 的基础上，后期找设计师重绘核心的 10-15 个高频图标（方案 D+E 混合）。
