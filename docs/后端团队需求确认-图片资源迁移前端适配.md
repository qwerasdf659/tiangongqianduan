## 二、需要后端确认的 API 变更（7 项）

### 确认项 1（高优先级）：广告投放接口 — AdDeliveryItem 响应格式

**接口**：`GET /api/v4/system/ad-delivery?slot_type=popup|carousel|announcement`

**前端期望的新响应格式**：

```json
{
  "success": true,
  "data": {
    "items": [
      {
        "ad_campaign_id": 1,
        "ad_creative_id": 5,
        "title": "春季活动",
        "content_type": "image",
        "primary_media_id": 42,
        "primary_media": {
          "media_id": 42,
          "object_key": "popup-banners/xxx.jpg",
          "public_url": "https://omqktqrtntnn.sealosbja.site/popup-banners/xxx.jpg",
          "mime_type": "image/jpeg",
          "width": 819,
          "height": 1092,
          "thumbnails": {
            "small": "https://..../popup-banners/xxx_150.jpg",
            "medium": "https://..../popup-banners/xxx_300.jpg",
            "large": "https://..../popup-banners/xxx_600.jpg"
          }
        },
        "display_mode": "tall",
        "link_url": null,
        "link_type": "none",
        "frequency_rule": "once_per_day",
        "..."
      }
    ]
  }
}
```

**需要确认**：
1. 后端 `GET /api/v4/system/ad-delivery` 是否已将 `image_url` / `image_width` / `image_height` 替换为 `primary_media_id` + `primary_media` 嵌套对象？
2. `content_type='text'`（系统公告）时，`primary_media` 是否为 `null`？
3. `display_mode` / `link_url` / `link_type` 是否仍然保留在响应的顶层（不放入 `primary_media` 内）？

**前端已做的改动**：
- `typings/api.d.ts` 中 `AdDeliveryItem` 接口已删除 `image_url` / `image_width` / `image_height`，新增 `primary_media_id` + `primary_media: MediaObject | null`
- `popup-banner.ts/wxml` 已从 `banner.image_url` 改为 `banner.primary_media?.public_url`
- `lottery.ts/wxml` 轮播图已从 `item.image_url` 改为 `item.primary_media?.public_url`
- 图片预加载已从 `banner.image_url` 改为 `banner.primary_media?.public_url`
