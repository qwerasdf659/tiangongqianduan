## 三、JWT Token 格式确认

前端 JWT 解析逻辑依赖以下 payload 字段，请后端确认 Token 格式一致：

```
JWT Payload 必需字段：
{
  "user_id": number,        // 用户ID
  "role_level": number,     // 角色等级（0=普通用户, 20=商家店员, 40=商家店长, 100=管理员）
  "exp": number,            // 过期时间戳（Unix秒）
  "iat": number             // 签发时间戳（Unix秒）
}
```

前端 Token 处理流程：

```
请求发送 → 检查 access_token 有效性（JWT完整性+过期时间）
    ↓ 过期
自动调用 POST /api/v4/auth/refresh（携带旧 access_token）
    ↓ 刷新成功
用新 access_token 重试原请求
    ↓ 刷新失败
清除认证数据 → 跳转登录页
```

**后端确认项：**

- [ ] refresh 接口是否接受过期的 access_token（前端当前实现：刷新时携带旧token）
- [ ] refresh 接口返回格式：`{ success: true, data: { access_token: "xxx", refresh_token: "xxx" } }`
- [ ] 401 错误码细分是否包含：`TOKEN_EXPIRED`、`SESSION_EXPIRED`、`SESSION_REPLACED`
