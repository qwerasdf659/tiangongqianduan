# 后端需求：兑换订单列表 — 资产名称中文化

## 问题描述

微信小程序"我的订单"页面（兑换订单列表）中，消耗资产显示为英文技术标识符，用户无法理解。

**当前显示效果（错误）：**
- 消耗: 30 star_stone
- 消耗: 100 red_core_shard

**期望显示效果（正确）：**
- 消耗: 30 星石
- 消耗: 100 红源晶碎片

## 涉及接口

| 项目 | 说明 |
|------|------|
| API 路径 | `GET /api/v4/exchange/orders` |
| 请求方式 | GET |
| 认证要求 | 需要 JWT Token |
| 前端调用位置 | `utils/api/backpack.ts` → `getExchangeRecords()` |
| 前端展示位置 | `packageExchange/exchange-orders/exchange-orders.wxml` 第133行 |

## 后端实现建议

在查询 `exchange_records` 表时，LEFT JOIN `asset_group_defs` 表，通过 `asset_code` 关联获取 `display_name`（或 `asset_name`）字段。

```sql
SELECT 
  er.*,
  agd.display_name AS pay_asset_name
FROM exchange_records er
LEFT JOIN asset_group_defs agd ON er.pay_asset_code = agd.asset_code
WHERE er.user_id = :user_id
ORDER BY er.created_at DESC
LIMIT :page_size OFFSET :offset;
```

> 注：具体表名和字段名请以后端数据库实际结构为准。核心逻辑是把 `asset_code`（如 `star_stone`）映射为中文显示名称（如"星石"）。

## 前端适配说明

后端返回 `pay_asset_name` 字段后，前端将修改 `packageExchange/exchange-orders/exchange-orders.ts` 第263行：

```typescript
// 修改前（当前代码）
_payInfo: order.pay_amount ? `${order.pay_amount} ${order.pay_asset_code || ''}` : '',

// 修改后（后端提供 pay_asset_name 后）
_payInfo: order.pay_amount ? `${order.pay_amount} ${order.pay_asset_name || order.pay_asset_code || ''}` : '',
```

## 优先级

中等 — 影响用户体验，用户看到英文技术标识符会困惑。
