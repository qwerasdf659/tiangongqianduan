### ✅ 问题3（已确认）：POINTS 资产在 material_asset_types 表中的状态

**后端确认结果**（2026-02-15 查询真实数据库）：

`POINTS` **已在 `material_asset_types` 表中配置**，前端文档中"未配置"的信息已过时。

| 字段 | 值 |
|------|-----|
| `asset_code` | `POINTS` |
| `display_name` | `普通积分` |
| `is_tradable` | `false`（不可交易） |
| `is_enabled` | `false`（已禁用） |
| `group_code` | `points` |

**结论**：
1. `POINTS` 已配置，`display_name` 为"普通积分"
2. `POINTS` **不允许**在交易市场上架（`is_tradable=false`）
3. `POINTS` 已禁用（`is_enabled=false`），不会出现在背包 `assets[]` 列表中
4. 前端无需额外处理 `POINTS` 的展示问题

## 七、WebSocket 实时推送对接指南（✅ 后端已实现）

> 📅 更新时间：2026-02-15
> ✅ 后端已完成全部实现，前端可直接对接

### 技术方案：Socket.IO（已确定）

后端使用 **Socket.IO v4.8.1**，前端**必须使用 Socket.IO 客户端**连接，**不能用原生 WebSocket**。

#### 1. 安装依赖

```bash
npm install weapp.socket.io
```

安装后在微信开发者工具执行 **"工具 → 构建 npm"**。

#### 2. 连接方式

```javascript
import io from 'weapp.socket.io'

const socket = io('https://omqktqrtntnn.sealosbja.site', {
  transports: ['websocket'],   // 只用 websocket（跳过 HTTP 轮询）
  path: '/socket.io',          // 后端路径（默认值）
  auth: { token: accessToken } // JWT Token 通过 auth 对象传（不拼在 URL 上）
})
```

> ⚠️ **不要用** `wx.connectSocket`，原生 WebSocket 无法连接 Socket.IO 后端。

#### 3. 后端已支持的事件列表

| 事件名 | 方向 | 数据格式 | 说明 |
|--------|------|---------|------|
| `connect` | 内建 | — | 连接成功 |
| `connect_error` | 内建 | `error.message` | 连接失败（含 "Authentication" = Token 无效） |
| `connection_established` | 后端→前端 | `{ user_id, is_admin, socket_id, server_time, timestamp }` | 连接建立确认 |
| `new_message` | 后端→前端 | `{ chat_message_id, content, sender_type, ... }` | 新聊天消息 |
| `notification` | 后端→前端 | `{ type, title, message, ... }` | 系统通知 |
| `product_updated` | 后端→前端 | `{ action, exchange_item_id, name, stock, status, timestamp }` | 商品变更通知 |
| `exchange_stock_changed` | 后端→前端 | `{ exchange_item_id, name, remaining_stock, changed_amount, reason, timestamp }` | 库存变更通知 |
| `session_closed` | 后端→前端 | `{ session_id, close_reason, closed_at, ... }` | 客服会话关闭 |
| `send_message` | 前端→后端 | `{ session_id, content, message_type }` | 发送聊天消息 |
| `message_sent` | 后端→前端 | `{ chat_message_id, session_id, timestamp }` | 消息发送确认 |
| `message_error` | 后端→前端 | `{ error, message, timestamp }` | 消息发送失败 |

#### 4. 前端代码示例

```javascript
import io from 'weapp.socket.io'

const socket = io('https://omqktqrtntnn.sealosbja.site', {
  transports: ['websocket'],
  auth: { token: wx.getStorageSync('access_token') }
})

socket.on('connect', () => console.log('WS连接成功'))
socket.on('connection_established', (data) => console.log('用户ID:', data.user_id))
socket.on('connect_error', (err) => {
  if (err.message.includes('Authentication')) { /* 跳转登录页 */ }
})
socket.on('new_message', (msg) => { /* 更新聊天界面 */ })
socket.on('notification', (data) => { /* 显示系统通知 */ })
socket.on('product_updated', (data) => { /* 刷新商品列表 */ })
socket.on('exchange_stock_changed', (data) => { /* 更新库存显示 */ })
socket.on('session_closed', (data) => { /* 提示会话已关闭 */ })

// 发送聊天消息
socket.emit('send_message', { session_id: 'xxx', content: '你好', message_type: 'text' })
```

#### 5. 不需要前端实现的功能（Socket.IO 已内建）

| 功能 | 说明 |
|------|------|
| 心跳保活 | 自动25秒间隔，不用写定时器 |
| 断线重连 | 自动重连（1-30秒退避），不用写重连逻辑 |
| JSON序列化 | 自动处理，不用手动 JSON.parse / JSON.stringify |
| 消息路由 | 按事件名 socket.on('xxx', cb)，不用 switch/case |

| admin_message | new_message | 后端统一事件名 |
| system_message | notification | 后端统一事件名 |
| send_message | socket.emit('send_message', ...) | ✅ 一致 |
| product_updated | product_updated | ✅ 一致 |
| exchange_stock_changed | exchange_stock_changed | ✅ 一致 |
