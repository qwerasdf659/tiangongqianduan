# 微信小程序 - 交易市场前端对接文档

> 更新日期：2026-02-24
> 术语统一：全项目已统一使用「交易市场」（原 C2C 交易），微信小程序前端叫「交易市场」，后端/管理后台已同步。

---

## 一、API 基础信息

| 项目 | 说明 |
|------|------|
| Base URL | `/api/v4/market` |
| 认证方式 | `Authorization: Bearer <JWT Token>` |
| 响应格式 | `{ success, code, message, data, timestamp, request_id }` |
| 幂等控制 | 写操作需传 `Idempotency-Key` 请求头 |
| 敏感操作 | 上架/购买/撤回/兑换需有效登录会话 |

---

## 二、核心 API 端点

### 2.1 市场浏览

#### 获取挂牌列表
```
GET /api/v4/market/listings
```

**查询参数：**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | number | 否 | 页码，默认 1 |
| limit | number | 否 | 每页数量，默认 20 |
| listing_kind | string | 否 | 挂牌类型：`item`（物品）/ `fungible_asset`（材料） |
| asset_code | string | 否 | 资产代码筛选 |
| item_category_code | string | 否 | 物品类目筛选 |
| asset_group_code | string | 否 | 资产分组筛选 |
| rarity_code | string | 否 | 稀有度筛选 |
| min_price | number | 否 | 最低价格 |
| max_price | number | 否 | 最高价格 |
| sort | string | 否 | 排序方式：`newest` / `price_asc` / `price_desc` |

**响应数据：**
```json
{
  "success": true,
  "data": {
    "products": [...],
    "pagination": { "page": 1, "limit": 20, "total": 100, "total_pages": 5 }
  }
}
```

---

#### 获取筛选维度
```
GET /api/v4/market/listings/facets
```

**响应数据：** 类目、稀有度、资产分组、挂牌类型的枚举列表。

---

#### 获取挂牌详情
```
GET /api/v4/market/listings/:market_listing_id
```

---

#### 获取结算币种
```
GET /api/v4/market/settlement-currencies
```

### 2.3 上架操作

#### 上架物品实例
```
POST /api/v4/market/list
```

**请求头：** `Idempotency-Key: <唯一键>`

**请求体：**
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| item_id | number | 是 | 物品实例 ID |
| price_amount | number | 是 | 挂牌价格 |
| price_asset_code | string | 是 | 结算币种代码 |
| condition | string | 否 | 物品状态描述 |

---

#### 上架可叠加资产（材料）
```
POST /api/v4/market/fungible-assets/list
```

**请求头：** `Idempotency-Key: <唯一键>`

**请求体：**

## 三、关键业务规则

### 3.1 资产交易限制
- **POINTS（系统积分）** 和 **BUDGET_POINTS（预算积分）** 永久禁止在交易市场交易
- 其他资产是否可交易由后端 `is_tradable` 字段控制
- 前端无需判断，后端会统一返回错误码 `ASSET_MARKET_BLACKLISTED`

### 3.2 上架数量限制
- 每个用户有上架数量上限（默认 10 个，管理员可调整）
- 通过 `GET /listing-status` 获取当前/上限/剩余

### 3.3 交易手续费
- 交易手续费由后端统一计算，前端展示即可
- 购买响应中包含 `gross_amount`（总价）、`fee_amount`（手续费）、`net_amount`（卖家实收）

### 3.4 担保码机制
- 实物类交易需要担保码确认收货
- 购买响应中 `requires_escrow_confirmation = true` 表示需要担保码
- 买家输入卖家提供的 6 位担保码完成交易

### 3.5 幂等性
- 所有写操作（上架/购买/撤回/兑换）必须传 `Idempotency-Key` 请求头
- 建议使用 `UUID v4` 或 `{user_id}_{timestamp}_{random}` 格式

## 五、统一响应格式

```json
{
  "success": true,
  "code": "SUCCESS",
  "message": "操作成功",
  "data": { ... },
  "timestamp": "2026-02-24T15:30:00.000+08:00",
  "request_id": "req_abc123"
}
```

错误响应：
```json
{
  "success": false,
  "code": "ASSET_MARKET_BLACKLISTED",
  "message": "该资产类型禁止在交易市场交易: 系统积分",
  "data": null,
  "timestamp": "2026-02-24T15:30:00.000+08:00",
  "request_id": "req_abc123"
}
```

## 六、常见错误码

| 错误码 | 说明 |
|--------|------|
| `ASSET_MARKET_BLACKLISTED` | 积分类资产禁止在交易市场交易 |
| `ASSET_NOT_TRADABLE` | 该资产类型未开启交易（数据库配置） |
| `LISTING_LIMIT_EXCEEDED` | 用户上架数量已达上限 |
| `INVALID_LISTING_STATUS` | 挂牌状态不合法（已售出/已撤回等） |
| `INSUFFICIENT_BALANCE` | 余额不足 |
| `IDEMPOTENCY_CONFLICT` | 幂等键冲突（重复请求） |
| `UNAUTHORIZED` | 未登录或 Token 过期 |
