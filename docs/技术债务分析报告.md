## 🔴 高优先级技术债务

### 1. TypeScript 类型安全形同虚设

**涉及文件**: `tsconfig.json`

`tsconfig.json` 开了 `strict: true` 但随即关闭了几乎所有严格检查：

```json
{
  "strict": true,
  "noImplicitAny": false,
  "noImplicitThis": false,
  "strictBindCallApply": false,
  "strictPropertyInitialization": false,
  "strictNullChecks": false,
  "useUnknownInCatchVariables": false
}
```

**实际影响**：

- `pages/` 目录下有 **136处** `any` 类型使用
- `utils/` 目录下有 **65处** `any` 类型使用
- 所有 MobX Store 的 action 函数都使用 `this: any`，完全丧失类型推导

**示例** (`store/user.ts` 第82-84行)：

```typescript
setLoginState: action(function (
  this: any,     // ← 类型完全丢失
  userInfo: UserInfo,
```

**📌 决策**：**一步到位** — 一次性开启 `strictNullChecks` 等全部严格检查，集中修复所有类型错误（预计 200-400 个）。同时为 Store 创建正确的 `this` 类型泛型，彻底消除 `this: any`。

const BASE_CONFIG = { business: { ... }, security: { ... } }

### 11. URL 查询参数手动拼接

**涉及文件**: `utils/api.ts`

多个 API 函数用字符串拼接方式构建 URL 查询参数，缺少统一的工具函数：

```typescript
// 此模式在 api.ts 中重复了至少 8 处
let url: string = `/assets/transactions?page=${page}&page_size=${page_size}`
if (asset_code) {
  url += `&asset_code=${asset_code}`
}
if (business_type) {
  url += `&business_type=${business_type}`
}
```

**存在的问题**：

- 参数值未经 `encodeURIComponent` 编码
- 拼接逻辑重复，容易出错
- 无法统一处理空值/undefined

**建议**：提取 `buildQueryString(params: Record<string, any>): string` 工具函数。
